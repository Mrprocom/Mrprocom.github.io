Tips and Frequently Asked Questions

This page covers some useful tips, answers to frequently asked questions and common problems.

Tips

  • Use the console to do quick calculations: Since the console can be used as a Lua shell, you can use it to calculate numbers whenever you are taking measurements. Just open the console, write your calculation there (for example, 2 * (12^2 + 24) / 6 - 14), and then press Enter. It will give you the answer, which is 42 for that example. Use the asterisk sign * for multiplication.
  • Use "C" to set temperatures in Celsius: If you want to set temperatures in Celsius using the PROP tool or using the console, you do not have to convert from Celsius to Kelvin; just add "C" at the end of your temperature value. Example: !set temp dust 100C.
  • Do not make laggy saves: When you are making saves, make sure they look nice and not laggy; laggy saves are annoying to run, so users might not find them appealing.
  • Use deco: Always use decoration and colours whenever possible because these are very attractive.
  • Give instructions using elements: If you want to explain your save using text, write using elements instead of signs, preferably with an element that disappears over time, like BRAY. Use Lua scripts to help you write using elements like the Texter script.
  • Learn from other saves: If you find a complicated save and you do not know how it works, try to investigate it and learn the concepts behind it to improve your skills.
  • Do not report saves using comments: If you find a save that breaks the rules, instead of talking about it in the comments, report the save without saying anything in the comments. This will inform the moderators and will prevent you from being harassed by the author.
  • Avoid conflicts: If someone is harassing you in the comment section, ignore them and report the save they insulted you in. Moderators will delete the offensive comments and it will appear as if nothing major happened. They might even ban that user if they think it is well-deserved.
  • Store backups of your save: Always keep backups of the save you are working on. Use stamps to save your progress for later or save it locally, which is better than saving it online; even if it is not published, some users can still search for saves using IDs, find yours and possibly steal it.
  • Take inspiration: You can use Google Images for inspiration if you are having problems designing something.
  • Update or re-upload your decent saves: If you think your save deserves getting to the Front Page, but is not receiving much attention, try updating it every 3 hours to get it to the first position in the recent saves tab (save bumping). You could even try removing it and re-uploading it later when more users are active.
  • Be friendly: Be polite and respectful and always follow the rules so that you do not get banned and other do not harass you.
  • Check before asking or suggesting: Before you ask a question or post a suggestion on the forum, make sure that it has not been suggested or asked before. You can check by searching it on the website using the search bar.
  • Know how to suggest elements: When you post a suggestion to be added to the game, make sure to only mention one or more related features and list detailed information about them including how they can be useful.
  • Do not call it LUA: "Lua" means "Moon" in Portuguese, so it is not an acronym and should not be written as LUA. This may sound stupid, but some people find it annoying and some will not take you or your knowledge in programming seriously if you do not write it correctly.

Frequently Asked Questions

Am I allowed to make multiple accounts?

Yes, although abusing multiple accounts for vote fraud or spam is against the rules, using multiple accounts to allow you to vote multiple times will result in a ban.

How do I use Powder Toy behind a proxy?

You can use the Powder Toy behind a proxy by specifying it at the command line, On Windows and Linux, this is simply a case of issuing the command:

powder-sse(2/3) proxy:<address>:>port>

where <address> is the address of the proxy and <port> is the port.
On a Mac, this is a bit more complicated, you must open a terminal and go to the directory with Powder.app in, and issue the command:

Powder.app/Contents/MacOS/powder-x proxy:<address>:<port>

An example command would be:

powder.exe proxy:wwwcache.school.ac.uk:3128

When I try to login, I get a "605 Timeout" error. How do I fix this?

This error is very general. It means that the Powder Toy cannot communicate with the server, usually because there is no internet connection, or the port used (port 80) has been intentionally blocked by an administrator. If you are trying to use the Powder Toy from within a corporate or school network, it is likely you will need to use a proxy. Another known cause is Firewall or internet filtering software that runs on the computer, like Windows Live Family Safety, if this is the case, the relevant proxy settings need to be acquired, or the software disabled.

I have registered on the website, but I cannot sign in.

If you are told your username or password is incorrect, it may be that you registered with a username that is not compatible with the Powder Toy. To keep usernames readable, only alpha-numeric, as well as hyphens - and underscores _ characters are allowed in usernames, any other characters in your username are replaced with a hyphen -. Try using your adjusted username to sign in. Other common issues are:

  • Your password may have characters that you cannot correctly enter into the Powder Toy, in which case, you should change your password on the website in your profile.
  • You may not have typed your username or password correctly (they are case sensitive).
  • Another rare but possible issue is that the username or password is too long to be entered in the game login screen.

What is TPT and FP? and what is TPT++?

TPT stands for The Powder Toy and FP stands for Front Page, which is the first page displayed in the game and on the website by default. Powder Toy was originally written in C, but now it is being rewritten in C++.

How do I become a moderator?

Better do not ask questions like that. Be a nice user, help others, make saves, program for TPT and maybe Simon will notice you.

My new avatar does not appear, how do I fix that?

It takes some time for avatars to be updated. You can try emptying your browser's cache, if that does not work, then you will have to wait for about a day.

How can I remove my account?

Sadly, you cannot remove your account because that would cause a lot of problems. What you can do is ask a moderator to hide all your posts, unpublish all your saves and permanently ban your account, but your account will still exist.

Someone is violating the rules, what should I do?

If you see anything that violates the rules, report it using the report button shown when you preview the save before opening it. Do not report using comments, moderators will not be able to see them. You can also report your own save if it has inappropriate or offensive tags or if something bad has happened in the comments section. If you are reporting a stolen save, make sure you mention the ID of the original save in your report.

Modding FAQ

How do I compile?

  • Windows: Compiling with SCons or Compiling with Visual Studio
  • Mac OS X: Compiling for OS X
  • Debian/Ubuntu: Compiling on Debian/Ubuntu
  • Everything else: Use a program called SCons and the above guide, and get the needed packages and libs for your system.

Where do I get the latest source?

The Powder Toy repository on GitHub.

Where can I get help for developing mods?

Either on IRC or in the Development Assistance section.

What do I do if I get a large error?

Do not post large errors on the forum or on IRC; that would be spam. Post it on Pastebin instead, which does not require registering a new account.

  • If you are using Git Bash or MinGW, it might be hard to copy the error. To copy it, right click the header and select Mark, then mark a rectangle shaped selection area by dragging your cursor across the screen. Pressing enter will copy what you selected.
  • You might also be asked to post config.log. Only do this if it errored when looking for libraries, and not once it got to the compiling step.

Are there any pages to give me pointers?

  • TPTElements
  • Coding Tutorial
  • Variables
  • Functions

How do I cross-compile?

  • To compile for Windows on Linux: Compiling for Windows on Linux
  • To compile for Mac on Linux: Compiling for Mac on Linux

How do I compile a legacy version?

Legacy versions can be used on very old computers that do not support SSE. You probably do not need to compile this separately, but if someone asks for it, you can compile using --no-sse. If using visual studio, you will have to find the SSE option in project settings and turn it off.

Anything else?

  • If you do not know C++, it is recommended to either learn it, or use Lua instead.
  • Lua guides:
    • Powder Toy Lua API
    • TPTElements Lua Editor
  • You can also ask questions in the Lua Scripting forum section.

Rejected Suggestions List

Magnetic effects on metals

It would require a new layer and some fancy magnetic field calculations, which is a lot of work, and we already have a lot of layers like pressure, velocity x and y, gravity, portals, wifi and more. Coding magnets and proper magnetic fields is also hard. Unless someone takes the initiative to do it, it is rejected.

Freezers and heaters that affect the temperature of particles in contact with it

There are already many ways to control the temperature of particles, like controlled reactions, using HSWC and using Game of Life elements. Static GOL elements like MAZE cool particles down and moving GOL elements like WALL heat particles up.

Movable solids that stay together, fall down due to gravity, rotate and bounce off other solids

It is very hard to code and the current engine does not support it. The best we can do is very glitchy and not what people want.

Element that glues powders together into solids

This is sort of like moving solids and is rejected for the same reasons.

Element that run Lua or console commands when sparked

Other than being very exploitable, Lua commands could be dangerous and contain viruses because Lua allows OS related stuff to be done. It also causes speed issues; the commands need to be saved like signs, which would increase the size of saves, which is precisely why signs are limited. It would also be difficult to modify the code of the game without breaking so many saves that use those types of elements.

Element that gives particles it touches all of its properties

The reason why this suggestion is rejected is similar to why elements that run console or Lua commands when sparked are rejected; it might break a lot of saves if changes were made to the code. It also adds extra complication to the game that only very advanced users would understand and the element would break extremely easily.

Implementing sounds

Particle simulators like TPT are not made in a way that would easily allow sound to work. All particles on the screen exist at once, so sound would have to come out of everything at one time. It is hard to imagine how every sound playing at once would work. Perhaps the best that can be done is some element that plays a note sound when sparked. Entire realistic sounds for a few random reactions would not sound good though.

3D Powder Toy

It is hard to code and it would make the game extremely slow. It can be imagined as having many opened Powder Toy windows at the same time; the FPS count of that stack would be very low. However, 3D Powder Toy was added as an April Fools joke, but not the kind you would expect. If you open the options menu, you will see an option to change the depth of the 3D anaglyph effect. It is recommended to set it to 3 and use 3D glasses.

Anything related to stickmen

Stickmen are not the main point of this game, and they are already very glitchy.

Bigger or resizable simulation sizes

It is not possible; the simulation has to be a fixed size and cannot be resized. If a larger size was needed, it would be required to distribute it as another executable or do some unwanted stuff. Saves would not be compatible between versions and some people can barely handle the size now. The number of pixels on the screen would change significantly if the dimensions of the simulation were changed, causing much more lag.

More or unlimited signs

It is a bad idea to have unlimited signs. People are not supposed to make text-only saves, that is why they are limited. If you want to give instructions, you could draw them using elements, the best way to do so is to use a Lua script like Texter, which can be used to write down text using any element.

Smaller walls

Walls are bad enough as they are in the code. Making them less than 4x4 would break things due to how the pressure and velocity grids use that spacing. You can use alternatives like DMND and FRME. TTAN can be used to block air pressure.

Adding signatures to forum posts

Signatures are useless. They would make the forum less clean and organised and would make pages much longer since they take up some space. Sometimes they even become longer than the actual post itself. Profiles are already enough for sharing personal information and websites.
