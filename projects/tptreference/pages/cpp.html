Element Properties

For an element template, check out TPTElements.

PropertyDescription
Identifier Identifier string in the form of Namespace_PT_ElementName.
Name The name of the element in-game. It is recommended not to exceed 4 characters.
Colour Colour of the element in hexadecimal 0xRRGGBB.
MenuVisible Whether the element is visible in any menu section. 0 for hidden, 1 for visible.
MenuSection The menu section where the element is placed in. Check out Menu Sections for possible values.
Enabled Whether the element can be used or created in any way. 0 for disabled, 1 for enabled. You should leave it as 1.
Advection How much the particle is affected by air. 0 for solids and up to 1 for other elements, negative used in ANAR and DEST.
AirDrag How much air the particle generates in the direction of travel. Negative values are used for negative pressure.
AirLoss How much the particle slows down moving air. Values are from 0 (maximum effect) to 1 (no effect).
Loss How much velocity the particle loses each frame. Values are from 0 (used in energy particles) to 1 (used in solids).
Collision Used to calculate the new velocity when the particle collides with another.
Gravity How fast the particle falls down. Negative numbers are used to make the particle float just like in gases.
Diffusion How much the particle wiggles. Gases have it above 0 so that they wiggle.
HotAir How much the particle increases pressure. Mostly used in gases and VENT/VACU.
Falldown How the particle moves. 0 for solids, gases and energy particles, 1 for powders and 2 for liquids.
Flammable How much the particle can burn. 0 to make it unflammable. The higher the number, the more flammable the particle is.
Explosive Possible values are 0 for non-explosive, 1 for explosive by fire. 2 for explosive by fire or pressure above 2.5.
Meltable Whether the element is meltable (1) or not (0).
Hardness How much the element is affected by acid. 0 for no effect, anything higher for more effect.
PhotonReflectWavelengths The colour photons change to when they reflect off of this element in hexadecimal 0xAARRGGBB.
Weight How the particle sinks. -1 for energy particles, 1 for gases, 2 for light, 0-49 for liquids, 50-99 for powders and 100 for solids.
Temperature The spawn temperature of the element in Kelvin. R_TEMP + 273.15 for room temperature (22° C).
HeatConduct How fast the particle conducts heat. 0 for no transfer, 255 for maximum transfer.
Description Description of the element. Shows up when you hover over the element in-game.
Properties The special properties of the element, each separated by a binary OR |. Check out Property Constants for possible values.
LowPressure Change the type of the element when its pressure is lower than this value. Default is IPL.
LowPressureTransition The type of the element to change to when it is lower than LowPressure. Default is NT.
HighPressure Change the type of the element when its pressure is higher than this value. Default is IPH.
HighPressureTransition The type of the element to change to when it is higher than HighPressure. Default is NT.
LowTemperature Change the type of the element when its temperature is lower than this value. Default is ITL.
LowTemperatureTransition The type of the element to change to when it is lower than LowTemperature. Default is NT.
HighTemperature Change the type of the element when its temperature is higher than this value. Default is ITH.
HighTemperatureTransition The type of the element to change to when it is higher than HighTemperature. Default is NT.
Update Address of the update function, which gets executed each frame, it is where reactions and behaviours are coded.
Graphics Address of the grahpics function. This is where special visual effects are coded.

Property Constants

These are the possible values used in Properties, you can have multiple properties, but they must be joined together with a binary OR |.

ConstantDescription
TYPE_PART Used in powders.
TYPE_LIQUID Used in liquids.
TYPE_SOLID Used in solids and other types of elements.
TYPE_GAS Used in gases.
TYPE_ENERGY Used in energy particles.
PROP_CONDUCTS Allows the element to conduct electricity. Requires PROP_LIFE_DEC
PROP_BLACK Makes the element absorb photons of any colour.
PROP_NEUTPENETRATE Allows neutrons to pass through the element.
PROP_NEUTABSORB Makes the element absorb neutrons
PROP_NEUTPASS Makes the particles of that element get scattered when neutrons touch it.
PROP_DEADLY Makes the element deadly towards stickmen and fighters.
PROP_HOT_GLOW Makes the element gradually glow in red when it approaches its max temperature before melting.
PROP_RADIOACTIVE Makes the element have a radioactive glow like URAN and PLUT. Also makes it deadly towards stickmen and fighters.
PROP_LIFE_DEC Makes the life value of that element get decreased by 1 each frame.
PROP_LIFE_KILL Kills the element when its life value is less than or equal to 0.
PROP_LIFE_KILL_DEC Makes the life value if that element get decreased by 1 each frame until it reaches 0 and dies. If it starts with 0, it will not.
PROP_SPARKSETTLE Allows EMBR to touch it without disappearing.
PROP_NOAMBHEAT Prevents particles from that element to transfer ambient heat.
PROP_DRAWONCTYPE Allows the ctype of that element to be changed when drawn on, like CLNE.
PROP_NOCTYPEDRAW Prevents elements with PROP_DRAWONCTYPE to have their ctype changed when drawn on with this element.

Variables

VariableDescription
pmap[y][x] Particle Map. pmap[y][x]&0xFF returns particle type at (x, y). pmap[y][x]>>8 returns the index.
sim->photons[y][x] Photon Map. Works exactly like Particle Map, but it is only used for energy particles.
sim->bmap[y/CELL][x/CELL] Block Map. Returns wall at (x, y). Walls are in 4x4 grides, which is why cooCELL/tr>
sim->emap[y/CELL][x/CELL] Electronics Map. Returns 1 if the conductive wall at (x, y) is conducting electricity. Walls are in 4x4 grides.
sim->pv[y/CELL][x/CELL] Returns the pressure at (x, y). Pressure is in 4x4 grides.
sim->hv[y/CELL][x/CELL] Returns the ambient heat at (x, y). Ambient Heat is in 4x4 grides.
sim->vx[y/CELL][x/CELL] Returns the velocity in the x direction at (x, y). Velocity is in 4x4 grides.
sim->vy[y/CELL][x/CELL] Returns the velocity in the y direction at (x, y). Velocity is in 4x4 grides.
parts[i] Returns the particle with the index of i. Property examples: parts[i].ctype parts[i].dcolour parts[i].pavg[1]
r Generally used to store pmap[y][x].
t Generally used to store the current particle type.

Functions

Create a particle at the given coordinates:

sim->create_part(p, x, y, t)
Arguments:
  • int p: How the particle is created. -1 will create a particle if the space is empty. -2 will create the particle from brush. -3 will ignore particles in the space and create it anyway. Any positive number will override the particle at that index.
  • int x: The x coordinates of the particle.
  • int y: The y coordinates of the particle.
  • int t: The type of the particle to create. Use a pre-defined value such as PT_DUST.

Delete a particle with the given index:

sim->kill_part(i)
Arguments:
  • int i: Index of the particle to delete.

Delete a particle at the given coordinates:

sim->delete_part(x, y, flags)
Arguments:
  • int x: The x coordinates of the particle.
  • int y: The y coordinates of the particle.
  • int flags: Not used a lot, leave it as 0.

Change the type of a particle:

sim->part_change_type(i, x, y, t)
Arguments:
  • int i: Index of the particle to change.
  • int x: The x coordinates of the particle to change.
  • int y: The y coordinates of the particle to change.
  • int t: The type of the particle to create. Use a pre-defined value such as PT_DUST.

Clear a rectangular area:

sim->clear_area(x, y, w, h)
Arguments:
  • int x: The x coordinates of the top-left corner of the rectangle.
  • int y: The y coordinates of the top-left corner of the rectangle.
  • int w: The width of the rectangle.
  • int h: The height of the rectangle.

Create a rectangular area with the given element:

sim->CreateBox(x1, y1, x2, y2, t, flags)
Arguments:
  • int x1: The x coordinates of the top-left corner of the rectangle.
  • int y1: The y coordinates of the top-left corner of the rectangle.
  • int x2: The x coordinates of the bottom-right corner of the rectangle.
  • int y2: The y coordinates of the bottom-right corner of the rectangle.
  • int t: The type of the particle to create. Use a pre-defined value such as PT_DUST.
  • int flags: Not used a lot, leave it as 0.

Create a line with the given element:

sim->CreateLine(x1, y1, x2, y2, c)
Arguments:
  • int x1: The x coordinates of the starting point of the line.
  • int y1: The y coordinates of the starting point of the line.
  • int x2: The x coordinates of the ending point of the line.
  • int y2: The y coordinates of the ending point of the line.
  • int c: The type of the particle to create. Use a pre-defined value such as PT_DUST.

Flood-fill an enclosed area with the given element (works like Ctrl+Shift+Click in-game):

sim->FloodParts(x, y, t, cm, bm, flags)
Arguments:
  • int x: The x coordinates of any point in the enclosed area.
  • int y: The y coordinates of the same point in the enclosed area.
  • int t: The type of the particle to create. Use a pre-defined value such as PT_DUST.
  • int cm: A value used in the code. Leave it as -1.
  • int bm: A value used in the code. Leave it as -1.
  • int flags: Not used a lot, leave it as 0.

Sets a property via flood-fill (works like the PROP tool):

sim->flood_prop(x, y, propoffset, propvalue, proptype)
Arguments:
  • int x: The x coordinates of any point in the enclosed area.
  • int y: The y coordinates of the same point in the enclosed area.
  • size_t propoffset: Use the return value of offsetof(Particle, property), replace property with the property you want to set such as ctype.
  • PropertyValue propvalue: The property value you want to set to. For this, declare PropertyValue value then assign the property value to value.Integer for integers, or value.Float for temperatures or velocities, or value.UInteger for dcolour.
  • StructProperty::PropertyType proptype: The value type you provided in the previous argument. Possible values: StructProperty::Integer StructProperty::Float StructProperty::Uinteger

Check a particle with the given type is inbetween two particles with the given indexes. Return 1 if it is, otherwhise return 0:

sim->parts_avg(i, ni, t)
Arguments:
  • int i: Index of the first particle.
  • int ni: Index of the second particle.
  • int t: The type of the particle to check if it is in-between. Use a pre-defined value such as PT_DUST.

SCons Command Line Flags

Note that --release --sse --sse2 --no-warnings are all enabled by default. Use --debugging --no-sse --wall to disable them.

Some of these options are also auto-detected. Your platform is detected so --win --lin --mac are not needed unless you are cross compiling. Some cross compilers are also auto-detected on Linux, so --tool may not be needed. --64bit and --32bit should not be necessary either.

Flag ExampleDescription
-j4 Parallel Compilation: Compiles in multiple threads, speeds up compiling if you have the same number of threads and cores.
--build-dir build/powder64 Build: Sets the build directory.
--win Windows: Targets the Windows platform.
--lin Linux: Targets the Linux platform.
--mac Mac OS X: Targets the Mac OS X platform.
--msvc MSVC: Compiles using the Microsoft Visual Studio Compiler.
--tool i586-ming32msvc- Tool: Prefix for toolchain executables used for cross compiling.
--beta Beta: Makes a beta build.
--save-version 89 Save Version: Defines save version.
--minor-version 1 Minor Version: Defines the minor version.
--build-number 281 Build Number: Defines the build number.
--snapshot Snapshot: Makes a snapshot build.
--snapshot-id 12345 Snapshot ID: Defines the snapshot ID.
--64bit 64-bit: Compiles a 64-bit (AMD64) version.
--32bit 32-bit: Compiles a 32-bit (i386) version.
--universal Universal: Compiles a universal (32 and 64 bit) binary. OS X only.
--no-sse No SSE: Compiles a legacy version for older computers.
--sse SSE: Compiles an SSE-enabled, and thus, non-legacy binary.
--sse2 SSE2: Enables SSE2 optimisations.
--sse3 SSE3: Enables SSE3 optimisations.
--native Native: Compiles with --march=native.
--release Release: Enables compiler optimization and makes a release build, but it slows down compiling.
--debugging Debugging: Enables Debug Options.
--static Static Linking: Reduces external library dependencies.
--opengl OpenGL: Uses OpenGL to render the interface.
--opengl-renderer OpenGL Particles: Uses OpenGL to render particles. It also turns on --opengl.
--renderer Renderer: Compiles the save renderer.
--wall Error on Warnings: Any compiler errors turn into warnings.
--no-warnings No Warnings: Ignore all compiler warnings.
--nofft No FFT: Do not use fftw3f for gravity.
--nolua No Lua: Disables all Lua scripting features.
--output MyMod.exe Executable Name: Create an executable with a different name.
