Powder Toy Mods and Lua Scripts

The difference between mods and Lua scripts is that mods are modified versions [of the] game, so the entire executable file is different, but Lua scripts are scripts coded in a programming language called Lua, they can be loaded in Powder Toy anytime without having to use a different executable file. Mods are harder to make than Lua scripts. To get a mod, you only have to download it and run the executable file in it, you cannot merge it with normal Powder Toy. To run Lua scripts, you have to download them, put them in your Powder Toy directory, open the console in the game and run dofile("<File Name>"), so for example, if the Lua script you want to load is named tptmp.lua, run this in the console:

dofile("tptmp.lua")

If the script name is named tptmp.lua, it will be run automatically everytime you open the game. It is recommended that you get the Script Manager Lua script under the name autorun.lua, this script manages all your Lua scripts, makes it easy to load and unload them and allows you to browse and download more Lua scripts. You can install it with one command, open the console in the game and run:

tpt.getscript(1, "autorun.lua", 1)

To browse Lua scripts made by the community, check out the Script Server.

Some version numbers were extracted from the game, some from the source code and some were estimated. A new post has to be made announcing a new update for it to be noticed and updated here.
